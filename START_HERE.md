# START HERE — How This Course Works

## The Philosophy
You don't learn to code by reading. You learn by BUILDING.

Every single day of this course, you're building toward one thing:
**a complete offensive security platform that YOU wrote from scratch.**

Week 1, Day 1 — you write your first variable. 
Week 26 — you write a kernel module.
But every step in between is connected. Nothing is throwaway.

## The Rules
1. **Type everything yourself.** Never copy-paste from solutions.
2. **Run your code after every change.** See what breaks.
3. **When you're stuck for 15 minutes, look at the solution.** Not before.
4. **Exercises marked `___` have ONE correct answer.** Find it.
5. **CHALLENGE exercises have MANY valid answers.** Be creative.
6. **Weekend projects have NO solution file.** You figure it out. Come back to me for review.

## The Structure
```
Each day:
  1. WARMUP    — 5-minute review exercise (uses yesterday's skills)
  2. LESSON    — Read the concept (10-15 min)
  3. EXAMPLES  — Run code, predict output (15 min)
  4. EXERCISES — Fill blanks + write functions (30-45 min)
  5. CHALLENGE — Harder problem, no hints (15-30 min)

Each weekend:
  PROJECT — Combine the whole week into one tool (2-4 hours)
  No solution file. You build it yourself. Ask me to review.
```

## The Progression
```
Weeks 1-3:   You think in Python. Variables, loops, functions, files.
Weeks 4-6:   You build network tools. Scanners, parsers, scrapers.
Weeks 7-9:   You build offensive tools. Exploits, encoders, fuzzers.
Weeks 10-12: You build in C. Memory, pointers, binary manipulation.
Weeks 13-18: You understand the OS. Kernel, drivers, RE, exploitation.
Weeks 19-26: You're dangerous. Rootkits, vuln research, 0days.
```

## The One Rule That Matters
If you can't write it without looking at the solution, you haven't learned it yet. Go back and do it again until you can.
